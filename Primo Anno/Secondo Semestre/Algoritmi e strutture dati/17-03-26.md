# Limiti asintotici
Sia un polinomio:

$P(k) = a_k n^k + a_{k-1}n^{k-1} + a_0 n^0 = \Theta(n^k)$: **prendo il grado più elevato**

Per il caso migliore, prendo $\Omega(n^k)$

Per il caso peggiore, prendo $O(n^k)$

# Esempi sui limiti asintotici significativi
- $4n^2-2n$: $O(n^2)$, $\Omega(n^2)$, $\Theta(n^2)$
- $log\space n$: $O(log\space n)$, $\Omega(log\space n)$
- $log\space n^k$: $O(log\space n)$ perchè $log\space n^k$ = $k\space log\space n$
- $n\space log\space n$: $\Theta
(log\space n)$
- $(n + a)^b$: $\Theta(n^b)$
- $42$: $\Theta(1)$

# Esempi di algoritmi iterativi
***1. Scrivere un algoritmo iterativo che, dati due vettori $V_1$ e $V_2$, conta quanti elementi di $V_2$ sono contenuti anche in $V_1$***

```swift
Int Count(Int v1[], Int v2[])
{
    Int count = 0;
    For i = 1 To Length(v2) 
    {
        Int j = 1;
        While j <= Length(v1) And v2[i] != v1[j]
        {
            j++;
        }

        If j <= Length(v1)
        {
            count++;
        }
    }

    Return count;
}
```

**Tempi di esecuzione:**

Per ora, poniamo `Length(v1) == Length(v2) == n`.

**Il caso generale è:**

$C + C \cdot n + C \cdot n + C \cdot \sum_{i = 1}^{n} t_{wi} + C \cdot \sum_{i = 1}^{n} t_{wi} + C \cdot n + C \cdot t_{if} + C$

Dove $t_{wi}$ è 1 se il while è vero all'iterazione $i$ del for, e $t_{if}$ è il numero di volte che l'if è vero.

Semplifichiamo: $2C + 3Cn + 2c \sum_{i = 1}^{n} t_{wi} + Ct_{if}$

**Vediamo i casi specifici:**

- Caso migliore (tutti gli elementi di $V_2$ sono uguali al primo elemento di $V_1$, quindi $\sum_{i = 1}^{n} t_{wi} = 0$)

    $t_m(n) = 2C + 3Cn + Cn = 2C + 4Cn = \Omega(n)$
- Caso peggiore ($V_2$ e $V_1$ non hanno elementi in comune, quindi $\sum_{i = 1}^{n} t_{wi} = n$ e $t_{if} = 0$)

    $t_p(n) = 2C + 3Cn + 2C \sum_{i=1}^{n} n = 2C + 3Cn + 2Cn^2 = O(n^2)$

- Caso medio ($\sum_{i = 1}^{n} t_{wi} = \frac{n}{2}$)

    $\Theta(\frac{n^2}{2}p)$

***2. Siano due array di n bit, che rappresentano due numeri binari. Eseguire l'operazione di somma in un terzo vettore di n+1 bit (per il riporto).***

```swift
Void Somma(Int a[], Int b[], Int c[])
{
    Int r = 0; /* Riporto */
    For i = Length(a) Down To 1
    {
        c[i + 1] = r + a[i] + b[i];

        If c[i + 1] <= 1
        {
            r = 0;
        }
        Else If c[i + 1] > 1
        {
            r = 1;
            c[i + 1] = c[i + 1] - 2;
        }
    }

    c[1] = r; /* Riporto dell'ultimo bit */
}
```



**Tempi di esecuzione:**

**Il caso generale è:**

$C + Cn + Cn + Cn + Cn + Ct_{if1} + 2Ct_{if2} + C = 2C + 4Cn + Ct_{if1} + 2Ct_{if2}$

**Vediamo i casi specifici:**

- Caso migliore ($t_{if2} = 0$, quindi $t_{if1} = n$, cioè non esiste `i` tale che `a[i] == b[i] == 1`)
    
    $2C + 5Cn = \Omega(n)$
- Caso peggiore ($t_{if2} = n$, cioè quando `a[i] or b[i] == 1` per ogni `i != 1`)
    
    $2C + 4Cn + 2Cn = O(n)$

In generale quindi, il tempo è $\Theta(n)$.

***3. Data una matrice n x n, scrivere un algoritmo che dica se la matrice &eacute; simmetrica***

```swift
Bool simmetrica(Int m[][], Int n)
{
    Int r = 1;
    Int c = r + 1;

    While (m[r][c] == m[c][r] And r <= n - 1)
    {
        c++
        If (c < n)
        {
            r++;
            c = r + 1;
        }
    }

    If (r == n)
    {
        Return True;
    }

    Return False;
}
```

**Tempo d'esecuzione:**
$T(n) = 3cn + 3Ct_w + 2 \sum_{i = 0}^{t_w} t_{if} = ct_{if2} + cf{if2}$

***4. Analizzare la seguente funzione:***

```swift
Int f(Int n)
{
    Int r = 0;
    For i = 1 To n - 1
    {
        For j = i + 1 To n
        {
            For k = 1 To j
            {
                r++;
            }
        }
    }

    Return r;
}
```

Qui abbiamo tanti loop for annidati, le cui variabili iteratrici dipendono l'una dall'altra. Partiamo dal loop pi&ugrave; interno:

`For k = 1 To j`:
- Eseguito $j$ volte, ovvero $i + 1$
- Questo for &egrave; dentro ad un altro loop, quindi: $\sum_{j = i + 1}^{n} j$
- Quel loop &egrave; dentro ad un altro loop: $\sum_{i = 1}^{n - 1}\sum_{j = i + 1}^{n}j = \sum_{i = 1}^{n - 1} n(i + 1)$

$\begin{aligned}
\sum_{i=1}^{n-1} \sum_{j=i+1}^{n} j &= \sum_{i=1}^{n-1} (\dots + \dots + \dots) = \sum_{i=1}^{n-1} \left( \sum_{j=1}^{n} j - \sum_{j=1}^{i} j \right) = \sum_{i=1}^{n-1} \left( \frac{n(n+1)}{2} - \frac{i(i+1)}{2} \right) \\
&= \sum_{i=1}^{n-1} \left( \frac{n^2}{2} + \frac{n}{2} - \frac{i^2}{2} - \frac{i}{2} \right) = \sum_{i=1}^{n-1} \frac{n^2}{2} + \sum_{i=1}^{n-1} \frac{n}{2} + \sum_{i=1}^{n-1} -\frac{i^2}{2} + \sum_{i=1}^{n-1} -\frac{i}{2} \\
&= \left( \frac{n^2}{2} (n-1) \right) + \left( \frac{n}{2} (n-1) \right) + \left( -\frac{1}{2} \sum_{i=1}^{n-1} i^2 \right) + \left( -\frac{1}{2} \sum_{i=1}^{n-1} i \right) \\
&= \left( \frac{n^2}{2} (n-1) \right) + \left( \frac{n}{2} (n-1) \right) - \frac{1}{2} \left( \frac{n(n+1)(2n+1)}{6} \right) - \left( \frac{1}{2} \frac{(n-1)n}{2} \right) \\
&\approx \frac{n^3}{2} + \frac{n^2}{2} - \frac{2n^3}{12} - \frac{n^2}{4} = \frac{n^3}{2} + \frac{n^2}{2} - \frac{n^3}{6} - \frac{n^2}{4} \approx \frac{n^3}{6} + \frac{n^2}{4} \approx n^3
\end{aligned}$

In genere, tre for-loop innestati, anche se hanno indici dipendenti tra di loro, portano il tempo di esecuzione a $n^3$.

# Ricorsione

Tutti gli algoritmi ricorsivi sono composti da due elementi:
- **Caso base**: stabilisce dei casi in cui il codice ritorna un valore costante, terminando la ricorsione
- **Passo ricorsivo**: l'algoritmo richiama se stesso

Ad esempio:

Scrivere un algoritmo che calcoli l'$n$-esimo numero fattoriale:
```swift
Int factorial(Int n)
{
    If (n == 1)
    {
        return 1;
    }

    Return n * factorial(n - 1);
}
```

Scrivere un algoritmo che ritorni l'$n$-esimo numero nella serie di fibonacci:
```swift
Int fibonacci(Int n)
{
    If (n <= 1)
    {
        Return 1;
    }

    Return fibonacci(n - 1) + fibonacci(n - 2);
}
```
