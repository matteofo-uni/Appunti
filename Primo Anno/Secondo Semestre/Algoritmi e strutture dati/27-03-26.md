# Divide et impera
Consideriamo un problema $P$:
1. **Divide:** divido $P$ in due o più sottoproblemi ($P_1, P_2, \dots, P_k$)
2. **Impera:** risolvo ogni sottoproblema **ricorsivamente**
3. **Combina:** le soluzioni $S_1, S_2, \dots, S_k$ vengono ricombinate per ottenere la soluzione finale.

La "divisione" degli input deve essere dipendente dalla loro dimensione ($n$)

# Merge Sort
Scrivere un algoritmo di sorting ricorsivo che usi la tecnica divide et impera: **Merge sort**
1. **Divide**: dato un array di dimensione $n$, lo divido in due parti ciascuna con $\lceil\frac{n}{2}\rceil$ e $\lfloor\frac{n}{2}\rfloor$ elementi.
2. **Impera**: ogni sottoarray è ordinato ricorsivamente. Mi fermo quando arrivo ad avere un solo elemento per divisione.
3. **Combina:** fondo insieme i due array ordinati.

L'algoritmo è detto **stabile** perchè elementi dello stesso valore che si trovano in certe posizioni dell'array reciprocamente confrontabili vengono mantenute anche nell'array ordinate.

È inoltre **non in loco**, in quanto utilizza un numero di variabili che cresce proporzionalmente all'input.

```swift
// Esegue la parte di combinazione
Void Merge(Int a[], Int pi, Int pm, Int pf)
{
    Int i1 = pi;
    Int i2 = pm + 1;

    // Array d'appoggio
    Int b[];
    Int ib = 1;

    While (i1 <= m And i2 <= pf)
    {
        If (a[i1] <= a[i2])
        {
            b[ib] = a[i1];
            i1++;
        }
        Else
        {
            b[ib] = a[i2];
            i2++;
        }

        ib++;
    }

    While (i1 <= m)
    {
        b[ib] = a[i1];
        i1++;
        ib++;
    }

    While (i2 <= pf)
    {
        b[ib] = a[i2];
        i2++;
        ib++;
    }

    For (i = pi To pf)
    {
        a[i] = b[i];
    }
}

Void MergeSort(Int a[], Int pi, Int pf)
{
    If (pi < pf)
    {
        Int pm = (pi + pf) % 2;
        
        MergeSort(a, pi, pm);
        MergeSort(a, pm + 1, pf);

        Merge(a, pi, pm, pf);
    }
}
```

**Tempi di esecuzione:**

Nella funzione Merge:
- `While (i1 <= m And i2 <= pf)` Può ordinare fino a metà dell'array. Il resto dei loop itera per gli elementi rimanenti, quindi in totale i tre while loop vengono eseguiti $n$ volte, distribuite tra i tre in base all'input.
- L'algoritmo è quindi asintoticamente $\Theta(n)$.

Nella funzione MergeSort:
- L'equazione di ricorrenza è:
  - Caso base (n = 1): Eseguo una sola istruzione
  - Caso generale: $2T(\frac{n}{2}) + 2 + n$
- Vediamo come semplificare:
  - $T(n) = 2T(\frac{n}{2}) + n = 2[2T(\frac{n}{4}) + \frac{n}{2}] + n = 2^2 T(\frac{n}{2^2}) + \frac{n}{2^1} + \frac{n}{2^0} = \cdots$
  - $= 2^k T(\frac{n}{2^k}) + \frac{n}{2^{k - 1}} + \frac{n}{2^{k - 2}} + \dots + \frac{n}{2^0}$
  - Cerco di porre $\frac{n}{2^k} = 1$ per avere il caso base: $n = 2^k \implies k = log_2 n$
  - Ora abbiamo $2^{log_2n} T(1) + \cdots \approx n \space log\space n (\Theta(n \space log\space n))$
