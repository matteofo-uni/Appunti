# La macchina RAM (Random Access Machine)
- Una macchina ipotetica che può accedere a qualsiasi indirizzo nella stessa quantità di tempo
- Il tempo di accesso non influisce nel calcolo dei tempi d'esecuzione
- Memoria illimitata (no swap etc.)
- Macchina monoprocessore, esecuzione puramente sequenziale

# Algoritmi di ordinamento
## Problema
- Input: Un array di dimensione n
- Output: Un array di dimensione n
- Relazione: l'array di input permutato per avere tutti gli elementi in ordine (`v[n] <= v[n + 1]`)

## Selection sort
Ad ogni iterazione scelgo l'elemento più piccolo, e lo inserisco nella posizione corrispondente.

```swift
Void SelectionSort(Int v[])
{
    /* Trovo il minimo */
    For j = 1 To Length(v) - 1
    {
        Int min = j;
        For i = j + 1 To Length(v)
        {
            If v[i] < v[min]
            {
                min = i;
            }
        }

        Int temp = v[i];
        v[i] = v[min];
        v[min] = temp;
    }
}
```

**Andiamo ad analizzare i tempi d'esecuzione:**
- ```For j = 1 To Length(v) - 1``` viene eseguito $n$ volte, dove $n$ è `Length(v)`. Diamo un nome alla lunghezza di questa istruzione: $C_1$.
- `Int min = j` ($C_2$) viene eseguito $n - 1$ volte.
- `For i = j + 1 To Length(v)` ($C_3$) viene eseguito $\sum_{i=1}^{n} i$ volte.
- `If v[i] < v[min]` ($C_4$) viene eseguito $\sum_{i=1}^{n-1} i$ volte.
- `min = i` ($C_5$) dipende dalla disposizione degli elementi dell'array. Si dice $t_{if}$ (true if, riferendosi all'if statement che precede l'istruzione)
- `Int temp = v[i];` ($C_6$) viene eseguito $n - 1$ volte.
- `v[i] = v[min]` ($C_7$) viene eseguito $n - 1$ volte.
- `v[min] = temp;` ($C_8$) viene eseguito $n - 1$ volte.

Il calcolo dei tempi d'esecuzione risulta molto complesso nonostante la semplicità dell'algoritmo:

$T(n) = C_1(n) + C_2(n - 1) + C_3(\sum_{i=1}^{n} i) + C_4(\sum_{i=1}^{n-1}) + C_5(t_{if}) + C_6(n - 1) + C_7(n - 1) + C_8(n - 1)$

**Andiamo a semplificare:**

Raggruppiamo le istruzioni che vengono eseguite lo stesso numero di volte ($C_2$, $C_6$, $C_7$, $C_8$) in $4C$:

$T(n) = C_1(n) + C_3(\sum_{i=1}^{n} i) + C_4(\sum_{i=1}^{n-1}) + C_5(t_{if}) + 4C(n - 1)$

I tempi delle istruzioni individuali ($C_x$) è irrilevante quando analizzo il tempo d'esecuzione:

$T(n) = C(n) + C(\sum_{i=1}^{n} i) + C(\sum_{i=1}^{n-1}) + C(t_{if}) + 4C(n - 1)$

Possiamo semplificare il tempo di esecuzione dei loop da `n` a `n - 1`, in quanto un solo loop in più non causa grandi cambiamenti nel tempo di esecuzione:

$T(n) = 5C(n - 1) + 2C(\sum_{i=1}^{n-1} i) + C(t_{if})$

Andiamo a semplificare la sommatoria $\sum_{i=1}^{n-1} i = (n + 1) \cdot \frac{n}{2} = \frac{n(n + 1)}{2}$

$T(n) = 5C(n - 1) + 2C(\frac{(n) \cdot (n + 1)}{2}) + C(t_{if}) \approx 5C(n) + C(n^2) + C(t_{if})$

**Andiamo a vedere i casi specifici:**

- Caso migliore (l'array è già ordinato): $T_{min}(n) = 5C(n) + C(n^2) \approx n^2$
- Caso peggiore ($t_{if} = \frac{n(n+1)}{2} \approx n^2$): $T_{max}(n) = 5C(n) + C(n^2) + C(\frac{n^2}{2}) \approx n^2$
- Caso medio: $n^2$

## Insertion sort
Divido la lista in due parti: una ordinata e una non ordinata. A ogni iterazione, analizzo un elemento della parte non ordinata, e lo inserisco nella posizione opportuna nella parte ordinata.

```swift
Void InsertionSort(Int v[])
{
    For j = 2 To Length(v)
    {
        Int k = v[j];
        Int z = j - 1;

        While z > 0 And k < v[z]
        {
            v[z + 1] = v[z];
            z--;
        }

        v[z + 1] = k;
    }
}
```

**Andiamo ad analizzare i tempi d'esecuzione:**

- `For j = 2 To Length(v)` viene eseguito $n - 1$ volte (ignoro l'ultimo test).
- `Int k = v[j]` e `Int z = j - 1` sono eseguiti $n - 1$ volte.
- `While z > 0 And k < v[z]`, `v[z + 1] = v[z];` e `z--` sono eseguite $\sum_{i = 2}^{n} t_{wj}$ volte.
- `v[z + 1] = k` viene eseguito $n - 1$ volte.

**Formula generale del tempo di esecuzione:**

$T(n) = C(n - 1) + 2C(n - 1) + 3C(\sum_{i = 2}^{n} t_{wj}) + C(n - 1) = 4C(n - 1) + 3C(\sum_{i = 2}^{n} t_{wj})$

**Andiamo a vedere il caso specifico:**
- Caso migliore (la lista è già ordinata): $4C(n - 1) \approx n$
- Caso peggiore (la lista è in ordine inverso): $4C(n - 1) + 3C(\sum_{j=2}^{n} j - 1) = 4C(n - 1) + 3C(\sum_{i=1}^{n-1} i) = 4C(n - 1) + 3C(n^2) \approx n^2$, cioè $O(n^2)$

# Limiti asintotici
Per semplificare, utilizziamo i **limiti asintotici** (superiori, inferiori, stretti, non stretti).

Data una funzione $f(n)$ che è asintoticamente non negativa (definitivamente), cerco una funzione $g(n)$ e una costante $C_1 > 0$ tale per cui, a partire da un certo $n_0$, $f(n) < C_1 \cdot g(n)$.

Data invece una costante $C_2 > 0$ e una funzione $h(x)$ tale per cui, a partire da $n_0$, $f(n) > C_2 \cdot h(x)$.

Pongo:

$O(g(n))$ = $\{ f(n) \mid \exists C > 0, n_0 \geq 0 \mid \forall n \geq n_0 \space f(n) \leq C \cdot g(n) \}$

$\Omega(g(n)) = \{ f(n) \mid \exists C > 0, n_0 \geq 0 \mid \forall n \geq n_0 \space f(n) \geq C \cdot g(n) \}$
